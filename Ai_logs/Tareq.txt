APK analysis

You provided a1_case1.apk, and it was unpacked/decompiled into apk_audit/a1_case1/ using apktool and jadx.
The app was identified as a simple local-login Android app with three main activities: MainActivity, Login, and Profile.
The initial static review found two notable issues:
MainActivity stores username and password in plaintext in credentials.txt.
Login.generateSessionToken() creates a 16-character session token using java.util.Random and stores it in SharedPreferences.
I initially classified the findings as:
High: plaintext credential storage.
Medium/High-relevant randomness issue: insecure session token generation with weak randomness.
A key nuance found later: the APK does not clearly show that the stored session token is actually checked before granting access to Profile, so any claim of full session hijack/impersonation needed to be stated carefully.
